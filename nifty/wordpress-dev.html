Nifty WordPress Standards: 2022

Minimum Browser Support

What browsers do we want to support? Modern CSS and JavaScript features makes development easier and more performant, but we can't take advantage of them if we need to cater towards older browsers.

My Recommendation: Safari 14 on iOS or later. No Internet Explorer considerations.

If we decide to support earlier browsers, we need to use fallbacks when possible, but this means code bloat and some loss of performance.

Modern Feature Support

Feature Minimum Browser Benefit
Intersection Observer Safari 12, No IE Better performance for scrolling triggers in JS
WebP Safari 16 on Desktop, Safari 14 on iOS, No IE Smaller images, faster loading time
CSS Grid iOS 10 Easier layouts

// Disable Emojis
function disable_wp_emojicons() {
  // all actions related to emojis
  remove_action( 'admin_print_styles', 'print_emoji_styles' );
  remove_action( 'wp_head', 'print_emoji_detection_script', 7 );
  remove_action( 'admin_print_scripts', 'print_emoji_detection_script' );
  remove_action( 'wp_print_styles', 'print_emoji_styles' );
  remove_filter( 'wp_mail', 'wp_staticize_emoji_for_email' );
  remove_filter( 'the_content_feed', 'wp_staticize_emoji' );
  remove_filter( 'comment_text_rss', 'wp_staticize_emoji' );
  // filter to remove TinyMCE emojis
  add_filter( 'tiny_mce_plugins', 'disable_emojicons_tinymce' );
}
add_action( 'init', 'disable_wp_emojicons' );
function disable_emojicons_tinymce( $plugins ) {
  if ( is_array( $plugins ) ) {
    return array_diff( $plugins, array( 'wpemoji' ) );
  } else {
    return array();
  }
}

CSS & JS Refactoring

We need to refactor our custom CSS and JavaScript to make use of latest technologies to minimize impact on page speed and performance. We also need to make them more modular so we don't have unnecessary features being added to sites.

Lazy Loading Images Only with Intersection Observer

<script>
    // Lazy loading for images & iframes.
    let lazyObserver = new IntersectionObserver(processImages, {rootMargin: "0px 0px 200px 0px", threshold: 0});
    function processImages(entries, observer) {
      entries.forEach(entry => {
          if(entry.isIntersecting){
        if (entry.target.hasAttribute("data-url")) {
          entry.target.src = entry.target.dataset.url;
                observer.unobserve(entry.target);
        }
          }
    })
    }
      document.querySelectorAll('img, iframe').forEach(img => { lazyObserver.observe(img) });
</script>

Lazy Loading Images and Background Images with Intersection Observer

<script>
    // Lazy loading for image tags, background images & iframes.
    // The class 'lazyload' is for background images specifically & should use 'data-bg-img'
    let lazyObserver = new IntersectionObserver(processImages, {rootMargin: "0px 0px 200px 0px", threshold: 0});
    function processImages(entries, observer) {
      entries.forEach(entry => {
          if(entry.isIntersecting){
        if (entry.target.hasAttribute("data-url")) {
          entry.target.src = entry.target.dataset.url;
                observer.unobserve(entry.target);
        } else if (entry.target.hasAttribute("data-bg-img")) {
          oldStyles = entry.target.getAttribute("style")
          entry.target.setAttribute("style", `background-image:url('${entry.target.getAttribute("data-bg-img")}');${oldStyles}`)
                observer.unobserve(entry.target);
            }
          }
    })
    }
      document.querySelectorAll('img, iframe, .lazyload').forEach(img => { lazyObserver.observe(img) });
</script>

There are multiple "stacks" we can build on. Can we keep them separate, or do we have to find a way to make a unified stack?

Potential stacks/contexts:

  • Speed
  • AMP
  • Dark Mode
  • ADA

The Speed Stack

  • WPEngine allows us to have them automatically strip out all HTML comments. Ex: https://naqvilaw.com/wp-admin/admin.php?page=wpengine-common&tab=site-settings

  • Combine all font loading into a single CSS file and minify it.
